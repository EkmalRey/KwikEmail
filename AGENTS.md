# Agents.md

Guidance for autonomous coding agents working in this repository.

## What this project is

KwikEmail is a self-hosted disposable email service:

- A **Cloudflare Worker** (`packages/backend`) receives mail via Email Routing, owns addresses, and stores the last 24h of mail in D1. Unknown envelope recipients are rejected.
- A **homelab frontend** (`packages/frontend`) is the only UI. It proxies the Worker API and archives every inbox into persistent Node 24 SQLite (`/data/kwikemail.sqlite`). It merges cloud (live) and local (archived) mail keyed by message ID, tagging each row `storage: "cloud" | "local"`.

Key invariants (do not break without explicit user approval):
- `.env` is optional Compose interpolation for `PORT` and `BIND_ADDRESS`; production credentials are collected by the setup UI and stored encrypted in `/data`.
- Fresh installs use temporary password `123456` and require replacement before setup.
- The configured bootstrap `API_KEY` is the **only** key allowed on `/api/keys` (idempotent seed by hash).
- D1 auto-purges mail older than 24h; the frontend keeps local copies independently (startup + hourly).
- Browser polls only the selected inbox every 10s while visible; server archival runs on its own schedule.

## Repository layout

```
packages/backend/    Worker: src/*.ts, migrations/0001_initial.sql, wrangler.toml
packages/frontend/   Node server.js proxy + public/ browser UI (app.js, styles.css, index.html)
docker/              backend.Dockerfile, frontend.Dockerfile
docker-compose.yml   `kwikemail` appliance service with a `/data` bind mount
.env / .env.example  optional host binding only; never commit `.env`
```

## Conventions

- **Minimal/ponytail style**: prefer stdlib, native platform features, and existing helpers. No unrequested abstractions, config-for-constants, or scaffolding "for later". One-line solutions win. Mark deliberate shortcuts with `// ponytail:` comments.
- **No comments unless asked.** Code should be self-explanatory.
- **Testing is TDD and mandatory here**: every behavior change ships with a test. Backend: `npm test` (node:test) + `npm run typecheck` (tsc). Frontend: `npm test` (node:test).
- **Frontend browser code is source-tested**: `public/app.js` and `public/styles.css` can't run under node, so `ui.test.js` / `server.test.js` assert on source patterns (regex over the file). Keep UI logic in small source-checkable forms (e.g. `STORAGE_ICON`, `tickCount`).
- **Icons**: inline SVG only (no icon library). Cloud = `#3b82f6`, local = `#16a34a`. See `STORAGE_ICON` in `public/app.js` and `.store-ico.*` in `styles.css`.
- **Storage tagging** lives in `packages/frontend/archive.js` (`tagStorage`) and is applied in `server.js` proxy on `GET /api/emails`. Don't move this logic into the browser.

## Commands

```bash
# Backend
cd packages/backend && npm ci && npm test && npm run typecheck && npm run db:migrate:local

# Frontend (Node 24 required)
cd packages/frontend && npm test

# Compose (repo root)
docker compose config --quiet && docker compose up -d kwikemail
```

## Verification before claiming done

Run the package's `npm test` and (`backend`) `npm run typecheck`. For Docker changes, `docker compose config --quiet`. For Worker changes, `npx wrangler deploy --dry-run`; the checked-in Wrangler config is local-only and production deployment is generated by the setup UI.

## Things to watch

- Request body limit in the frontend proxy is 1 MiB (`archive.js`/`server.js` `readBody`); keep it.
- Proxy timeout to the Worker is 65s (`AbortSignal.timeout(65_000)`) to cover the 60s `/wait` endpoint.
- Migrations: add to `packages/backend/migrations/` and update `0001_initial.sql` naming; failures must stop startup.
- `npm ci` (not `npm install -g`) for Wrangler in images; Wrangler is pinned locally.
